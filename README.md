## 📌 Project Overview
This project is an end-to-end data analytics solution designed to support daily business decision-making. It bridges the gap between raw backend data and intuitive front-end visualization. By processing historical sales and incoming supply data, this project extracts actionable insights regarding revenue trends, customer segmentation, and supply chain health.

## 🎯 Business Objectives
1. **Ad-Hoc Sales Analysis:** Monitor revenue trends and identify top-selling products to optimize marketing strategies.
2. **Customer Segmentation:** Identify key accounts (top buyers) to support customer loyalty programs.
3. **Inventory Management:** Compare supply inflow against sales outflow to prevent out-of-stock or overstock scenarios.

## 🛠️ Tech Stack & Methodology
* **Data Extraction & Cleaning:** Python (`pandas`) in Google Colab.
* **Data Transformation:** Replicating complex SQL operations (e.g., `JOIN`, `GROUP BY`, `Aggregations`) using Python to merge sales and supply datasets.
* **Data Visualization (BI):** Google Looker Studio for creating a responsive, user-centric, and interactive dashboard interface.

## 📈 Key Actionable Insights
* **Key Account Concentration:** A significant portion of revenue is generated by a specific group of top buyers. *Recommendation:* Implement a Key Account Management (KAM) program to maintain loyalty and offer volume-based discounts.
* **Inventory Depletion Alert:** Certain high-demand products (e.g., "BERAS") show an imbalance between total sales and incoming supply. *Recommendation:* Trigger early restocking protocols for these top-tier commodities to prevent lost sales.
